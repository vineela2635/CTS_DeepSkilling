# SOLID Principles

## Overview

SOLID is a set of five object-oriented design principles that help developers create maintainable, scalable, and loosely coupled software.

## Principles

### S - Single Responsibility Principle (SRP)

A class should have only one responsibility and one reason to change.

### O - Open/Closed Principle (OCP)

Software entities should be open for extension but closed for modification.

### L - Liskov Substitution Principle (LSP)

Objects of a subclass should be replaceable with objects of their superclass without affecting program correctness.

### I - Interface Segregation Principle (ISP)

Clients should not be forced to depend on interfaces they do not use.

### D - Dependency Inversion Principle (DIP)

High-level modules should depend on abstractions rather than concrete implementations.

## Learning Outcome

Implemented and executed practical Java examples for all five SOLID principles to understand how they improve software design, maintainability, flexibility, and scalability.

## Conclusion

Applying SOLID principles helps build clean, reusable, and maintainable software systems while reducing coupling and increasing code quality.
